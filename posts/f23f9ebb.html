Sqoop

Apache Sqoop 是一个使用 MapReduce 非常高效地在 Hadoop 集群和关系数据库之间传输数据的工具。它通过在多个数据节点上生成任务来并行下载数据的各个部分来工作。完成后，每条数据都会被复制以确保可靠性，并在集群中分布以确保您可以在集群上并行处理这些数据。

在此 需要找到一种工具，可以轻松地将结构化数据从 RDBMS 传输到 HDFS，同时保留结构。这使我们能够查询数据，但不会干扰或破坏数据上的任何常规工作负载。

Apache Sqoop 是 CDH 的一部分，就是那个工具。Sqoop 的好处是我们可以自动将我们的关系数据从 MySQL 加载到 HDFS，同时保留结构。

通过一些额外的配置参数，我们可以更进一步，将这些关系数据直接加载到一个可供 Impala（CDH 附带的开源分析查询引擎）查询的表单中。鉴于我们可能希望将 Apache Avro 文件格式的强大功能用于集群上的其他工作负载（因为 Avro 是一种 Hadoop 优化的文件格式），我们将采取一些额外的步骤来使用 Avro 文件格式将此数据加载到 Impala ，因此它很容易用于 Impala 以及其他工作负载。

sqoop import-all-tables \
    -m 1 \
    --connect jdbc:mysql://quickstart:3306/retail_db \
    --username=retail_dba \
    --password=cloudera \
    --compression-codec=snappy \
    --as-parquetfile \
    --warehouse-dir=/user/hive/warehouse \
    --hive-import

这个命令可能需要一段时间才能完成，但它做了很多。它正在启动 MapReduce 作业以从我们的 MySQL 数据库中提取数据并将数据写入 HDFS，以 Apache Parquet 格式分布在整个集群中。它还创建表来表示 Impala / Apache Hive 中具有匹配架构的 HDFS 文件。

Parquet 是一种为 Hadoop 上的分析应用程序设计的格式。它不像典型的数据格式那样将数据分组到行中，而是将数据分组到列中。这对于许多分析查询是理想的，在这些查询中，您不是从特定记录中检索数据，而是分析许多记录中特定变量之间的关系。Parquet 旨在优化这些场景中的数据存储和检索。

命令完成后，我们可以确认我们的数据已导入 HDFS：

hadoop fs -ls /user/hive/warehouse/
hadoop fs -ls /user/hive/warehouse/categories/

注意： 显示的 .parquet 文件的数量将等于 Sqoop 使用的映射器的数量。在单节点上，您只会看到一个，但更大的集群将拥有更多的文件。

Hive 和 Impala 还允许您通过使用“CREATE EXTERNAL TABLE”语句在现有文件上定义模式来创建表，类似于传统的关系数据库。但是 Sqoop 已经为我们创建了这些表，所以我们可以继续查询它们。

我们将使用 Hue 的 Impala 应用程序来查询我们的表。Hue 为 CDH 中的许多工具提供了一个基于 Web 的界面，可以在管理器节点的端口 8888 上找到 （这里）。
在 QuickStart VM 中，Hue 的管理员用户名是“cloudera”，密码是“cloudera”。

进入 Hue 后，单击 Query Editors，然后打开 Impala Query Editor。

为了在查询期间节省时间，Impala 不会不断轮询元数据更改。所以我们必须做的第一件事就是告诉 Impala 它的元数据已经过时了。然后我们应该看到我们的表出现，准备好被查询：

# 使元数据无效；

invalidate metadata;


# 显示表格；
show tables;

刷新数据

牛刀小试：解决 DataCo 的业务问题了。SQL 示例查询，用于计算每个产品的总收入并显示前 10 名创收产品：

-- 最受欢迎的产品类别
select c.category_name, count(order_item_quantity) as count
from order_items oi
inner join products p on oi.order_item_product_id = p.product_id
inner join categories c on c.category_id = p.product_category_id
group by c.category_name
order by count desc
limit 10;

清除之前的查询，并将其替换为以下内容：

-- 10 大创收产品

select p.product_id, p.product_name, r.revenue
from products p inner join
(select oi.order_item_product_id, sum(cast(oi.order_item_subtotal as float)) as revenue
from order_items oi inner join orders o
on oi.order_item_order_id = o.order_id
where o.order_status <> 'CANCELED'
and o.order_status <> 'SUSPECTED_FRAUD'
group by order_item_product_id) r
on p.product_id = r.order_item_product_id
order by r.revenue desc
limit 10;
