HiveQL是Hive查询语言。和普通使用的所有SQL方言一样，Ta不上网安全遵守任何一种ANSISQL表针的修订版。HiveQL和MySQL的方言最相似，但是两者还是存在显著性的差异。

Hive不支持 行级

  • 插入操作、
  • 更新、
  • 删除操作。

Hive也不支持事物。Hive增加了在Hadoop北京下的可以提供跟高性能的扩展，一级一些个性化的扩展，甚至还增加一些外部程序。

数据库操作

创建数据库

create database jack;

如果数据库jack已经存在的话，将会抛出一个错误信息，使用以下语句可以避免在这中情况下抛出错误信息

create database if not exists jack;

查看数据库

show databases;

删除数据库

drop database jack;

进入数据库

use jack

Hive表操作

Hive表分类

  • 管理表（普通表）
  • 外部表

管理表

管理表中的数据存储有配置项 hive.metastore.warehouse.dir (默认,/user/hive/warehouse)所定义的目录的子目录下。当删除一个管理表时，Hive也会删除这个表中的数据。

创建表

create table user_info(
    id int,
    name string,
    ds string,
    days string
)
ROW FORMAT DELIMITED 
FIELDS TERMINATED BY '\t' 
STORED AS TEXTFILE;

## 同样可以避免重复创建报错
create table if not exists user_info(
    id int,
    name string,
    ds string,
    days string
)
ROW FORMAT DELIMITED 
FIELDS TERMINATED BY '\t' 
STORED AS TEXTFILE;

查看表


show tables;

# 查看表属性

show tblproperties user_info;

外部表

数据分区目的是为了水平分散压力，将数据从物理上转移到和最频繁的用户更近的地方，以及实现其他目的。

Hive中有分区的概念。分区变改变了Hive对数据存储的组织方式，反映在分区结构的子目录，分区字段一旦创建好，与其普通的字段一样，在使用分区时，用WHERE子句中过滤分区字段即可

## 创建分区表
CREATE TABLE user_id(
    id string ) PARTITIONED BY (day_id string);


## 导入数据(服务器本地)


load data local inpath '/home/cloudera/databricks/ids.txt' overwrite into table user_id partition (day_id='2021');


## 查看分区数据

select * from user_id where day_id='2021' ;

外部分区表

外部表同样可以使用分区，事实上，用户可能发现，管理大型生产数据集最常用的情况。这种结合给用户提供了一个可以行和其他工具共享数据的方式，同事也可以优化查询性能。

因为用户可以自己定义目录结构，因此用户对于目录结构的使用具有更多的灵活性。

CREATE EXTERNAL TABLE user_info_extenal(
    id string, name string)  PARTITIONED BY ( it_Id string) 
    ROW FORMAT DELIMITED
    FIELDS TERMINATED BY '\t' ;

外部分区表添加数据

hive修复分区


alter table user_info_extenal add partition(it_Id='2021') location '/user/jack/user_info/ ;

删除表

Hive支持和SQL中DROP TABLE 命令类似的操作。

drop table user_info;

可以选择是否使用IF EXITST关键字。如果没有使用这个关键字而且表并不存在的话，那么将会抛出一个错误的信息。

  • 对于管理表，表的元数据信息和表内的数据都会被删除，而外部表只删除元数据信息。

  • 修改表大多数的表属性可以通过ALTER TABLE 语句来进行修改。这种操作会修改元数据，但不会修改数据本身。

表重命名

alter table user_id rename to user_ids;

修改表属性

alter table user_info set tblproperties ( 'user_info'='databricks');

修改存储属性

alter table user_ids partition ( day_id='2021') set fileformat rcfile;



alter table user_ids partition ( day_id='2021') set fileformat textfile;

表分区操作

查看分区

show partitions user_ids;

增加分区

alter table user_ids add partition(day_id='2022');

修改分区

alter table user_ids partition(day_id='2022') rename to partition(day_id='2020');

删除分区


alter table user_ids add partition(day_id='2022');

alter table user_ids drop partition(day_id='2022');

列操作

修改列

注意： 列操作的 数据类型为必填。


alter table tablename change column column_orign column_new int(修改后列的属性) comment 'column_name'


alter table jack.itdocs change column `_c2` age int;

增加列

alter table user_info add columns (phone  string );

删除或替换列

alter table user_info replace columns ( phone string );

Hive建表高阶

CTAS

CTAS – as select方式建表（会复制表结构和表数据）


create ctas_employee as select * from jack.ds;

注：CTAS不能创建partition, external, bucket table

CTE

(CTAS with Common Table Expression)

create table jobs_detail as
with
tmp as (select jobs from employee_partition where info.gender='Male'),
t2 as (select jobs from employee_partition where info.gender='Female')
select tmp.jobs male_job,t2.jobs female_job from tmp,t2;

Like

（只复制表结构）

create table databricks_like like databricks;
